#### گواهی‌های خودامضا

اسکریپت چهار روش گواهی ارائه می‌دهد:

1. **ACME HTTP Challenge** (توصیه می‌شود) — نیاز به باز بودن پورت TCP 80 دارد
2. **فایل‌های گواهی محلی** — از فایل‌های cert و key خود استفاده کنید
3. **گواهی خودامضا** (پیش‌فرض) — گواهی برای هر دامنه تولید می‌کند، از طریق اثر انگشت pinSHA256 تأیید می‌شود (امن، بدون نیاز به دامنه/پورت)
4. **ACME DNS Challenge** — برای اعتبارسنجی مبتنی بر DNS

**امنیت گواهی خودامضا:** از نسخه ver1.12، گواهی‌های خودامضا به طور پیش‌فرض از تأیید اثر انگشت pinSHA256 استفاده می‌کنند. کلاینت‌ها به جای تکیه بر زنجیره‌های اعتماد CA، اثر انگشت گواهی سرور را تأیید می‌کنند. این روش هم امن و هم راحت است — نیازی به دامنه واقعی یا پورت‌های باز نیست.

**پیکربندی کلاینت:**
- کلاینت اصلی Hysteria2: از فیلد tls.pinSHA256 استفاده می‌کند
- کلاینت mihomo: از فیلد fingerprint استفاده می‌کند
- کلاینت sing-box: گواهی CA را از طریق فیلد tls.certificate جاسازی می‌کند
