#### Самоподписанные сертификаты

Скрипт предоставляет четыре метода сертификации:

1. **ACME HTTP Challenge** (рекомендуется) — требует открытого TCP порта 80
2. **Локальные файлы сертификатов** — используйте собственные файлы cert и key
3. **Самоподписанный сертификат** (по умолчанию) — генерирует сертификат для любого домена, проверяется через отпечаток pinSHA256 (безопасно, не требует домена/порта)
4. **ACME DNS Challenge** — для проверки на основе DNS

**Безопасность самоподписанного сертификата:** Начиная с ver1.12, самоподписанные сертификаты по умолчанию используют проверку отпечатка pinSHA256. Клиенты проверяют отпечаток сертификата сервера, а не полагаются на цепочки доверия CA. Это одновременно безопасно и удобно — не нужен реальный домен или открытые порты.

**Конфигурация клиента:**
- Нативный клиент Hysteria2: использует поле tls.pinSHA256
- Клиент mihomo: использует поле fingerprint
- Клиент sing-box: встраивает сертификат CA через поле tls.certificate
