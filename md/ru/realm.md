#### Режим Realm (P2P пробив)

Режим Realm позволяет запускать сервер Hysteria2 без публичного IP и проброса портов.

### Как это работает
1. Сервер и клиент регистрируются на сервере rendezvous
2. Сервер rendezvous помогает обеим сторонам обнаружить адреса друг друга
3. UDP-пробив устанавливает прямое соединение
4. Трафик идёт напрямую между сервером и клиентом (не через сервер rendezvous)

### Требования
- Обе стороны должны иметь UDP-связность
- STUN-серверы помогают обнаружить публичные NAT-адреса
- Лучше всего работает с full-cone NAT; симметричный NAT может потребовать дополнительных техник

### Конфигурация
Скрипт поддерживает как официальный сервер rendezvous (`realm.hy2.io`), так и собственные серверы rendezvous.

**Официальный rendezvous:** Токен по умолчанию — `public`, изменений не требуется.

**Формат адреса Rendezvous:** `realm://<token>@<rendezvous-host>/<realm-name>`

### Интеграция с Cloudflare WARP
Режим Realm может работать с Cloudflare WARP для скрытия реального IP сервера:
1. Установите WARP на сервер
2. Hysteria2 использует WARP IP для Realm-пробива
3. Клиенты подключаются через границу Cloudflare, скрывая реальный IP сервера
4. Фактически Hysteria2 работает за CDN Cloudflare

### Конфигурация клиента
- **Нативный клиент Hysteria2:** использует URI realm напрямую как поле server
- **mihomo:** использует блок realm-opts (enable/server-url/token/realm-id/stun-servers)
- **sing-box:** использует блок realm (server_url/token/realm_id/stun_servers), требуется 1.14+

### Совместимость с NAT
- Сопоставление портов UPnP/NAT-PMP включено по умолчанию
- ipMode поддерживает dual/ipv4/ipv6 (по умолчанию: dual)
