## Включение маскировочного веб-сайта

Hysteria2 поддерживает три режима маскировки для скрытия вашего прокси-трафика:

### 1. Строка (по умолчанию)
Возвращает фиксированный строковый ответ. Просто и легко.

### 2. Прокси (обратный прокси)
Работает как обратный прокси, предоставляя контент с другого веб-сайта. Контент удалённого сайта ретранслируется через ваш сервер.

Конфигурация: введите URL целевого веб-сайта (например, `https://www.example.com`)

### 3. Файл (статический файловый сервер)
Предоставляет статические файлы из локальной директории. Директория должна содержать файл `index.html`.

Конфигурация: укажите путь к локальной директории с вашим статическим сайтом.

### Дополнительные опции
- **TCP-маскировка:** Также слушайте TCP-порт для усиления маскировки. Когда браузер посещает `https://your-domain:port`, он видит маскировочный контент.
- **Блокировка HTTP/3:** Опционально заблокируйте UDP/443, чтобы QUIC-соединения не обходили ускорение Hysteria2. Рекомендуется для лучшего опыта с YouTube.

### Настройка
Все режимы маскировки поддерживают:
- Пользовательские заголовки ответа
- Пользовательские коды состояния
- Пользовательское содержимое тела ответа
