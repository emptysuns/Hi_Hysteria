## 1. Введение

> Hysteria2 — это многофункциональный сетевой инструмент, оптимизированный для неблагоприятных сетевых условий (двустороннее ускорение) — спутниковые каналы, перегруженный публичный Wi-Fi, **подключение к зарубежным серверам из Китая** и т.д. На базе модифицированного протокола QUIC.
> 
Он решает главную проблему при настройке прокси-серверов — **плохое качество сетевых маршрутов**.

1. China Telecom напрямую к NTT Япония + Cloudflare WARP, неоптимизированная магистраль 163, тест скорости в часы пик (20:00-23:00):

## 2. Возможности

<details>
<summary><b>Нажмите, чтобы развернуть полный список функций</b></summary>

* Поддержка всех трёх режимов маскировки Hysteria2 с широкими возможностями настройки
* Четыре способа импорта сертификатов:
  * ACME HTTP Challenge
  * ACME DNS
  * Самоподписанный сертификат для любого домена
  * Локальные файлы сертификатов
* Просмотр статистики сервера Hysteria2 в SSH-терминале:
  * Статистика трафика пользователей
  * Количество онлайн-устройств
  * Активные соединения и прочее
* Правила маршрутизации доменов на основе ACL с поддержкой блокировки
* Поддержка всех основных операционных систем и архитектур:
  * ОС: Arch, Alpine, RHEL, CentOS, AlmaLinux, Debian, Ubuntu, Rocky Linux и др.
  * Архитектура: x86_64, i386/i686, aarch64/arm64, armv7, s390x, ppc64le
* Генерация QR-кодов для ссылок hy2 прямо в терминале — меньше копирования
* Генерация оригинальных конфигурационных файлов Hysteria2 с полным набором параметров
* Запуск процесса Hysteria2 с высоким приоритетом — скорость прежде всего
* Прыжки портов и демон Hysteria2 управляются через скрипты автозапуска для лучшей расширяемости
* Скрипт установки Hysteria v1 сохранён как опция
* Расчёт BDP (произведение задержки на пропускную способность) для настройки параметров QUIC
* Поддержка исходящего SOCKS5, включая автоматическую настройку WARP
* Поддержка LXC, OpenVZ, KVM и всех основных платформ виртуализации
* Режим Realm (P2P пробив) — установка соединений без публичного IP и проброса портов
* Realm через Cloudflare WARP — подключение к Hysteria2 через WARP IP, скрытие реального IP сервера (фактически за CF CDN)
* Своевременные обновления — адаптация в течение 24 часов после релизов Hysteria2

## 5. Заключение

Hysteria2 хорошо работает в средах с высокой задержкой и потерей пакетов благодаря собственному алгоритму контроля перегрузки Brutal.

Это вносит вклад в исследование сложных сетевых сред. Цель этого репозитория — помочь исследователям удобно настраивать Hysteria2. В принципе, мы поддерживаем пользовательскую конфигурацию для всех функций Hysteria2, предоставляя широкие возможности настройки.

Если это помогло вам в изучении shell-скриптов, пожалуйста, поставьте ⭐ этому репозиторию, чтобы больше людей его обнаружили.

**Пожертвования и рекламное спонсорство не принимаются. Пожалуйста, не тратьте возможности issue.**
