# i18n 多语言支持 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `server/install.sh` 与 `server/hy2.sh` 增加中文、英语、波斯语、俄语多语言支持，并在 install 阶段让用户选择语言并下载对应翻译包。

**Architecture:** 新增 `server/i18n/` 目录存放 JSON 翻译文件（`en.json`/`zh.json`/`fa.json`/`ru.json`）。`install.sh` 负责选择语言并下载主脚本与同语言包；`hy2.sh` 内嵌轻量 i18n 加载器 `i18n <key> [args...]`，按“用户语言→英文→key 本身”链回退。语言文件声明 `schema_version`，脚本内置当前 schema，若版本不匹配则静默尝试更新，失败仍用英文 fallback。

**Tech Stack:** bash 4+/POSIX-ish, curl/wget, yq 用于 JSON？**不允许引入 yq 作为 i18n 依赖**（避免循环依赖 i18n 功能）。因此 i18n loader 必须纯 bash + `grep`/`sed` 实现，或要求依赖 `jq`（优先）并提供最小 fallback。最终选择：**无外部 JSON parser 依赖，使用轻量级 key-value 行读取器**，JSON 文件按 `"key": "value"` 每行存储（人类可编辑、便于 grep）。

## Global Constraints

- 新增依赖越少越好；i18n loader 不能依赖 yq/jq（脚本运行环境不可控）。
- fallback 链：用户语言（如 `zh`）→ 英文 `en` → raw key。
- 波斯语为 RTL，JSON 中标记 `rtl: true`，必要时包裹 RTL 控制字符。
- 语言文件必须和脚本同版本 schema；脚本内置 `HIHY_I18N_SCHEMA=1`。
- 用户可通过环境变量 `HIHY_LANG=xx` 或命令行 `--lang=xx` 覆盖选择，未指定才交互选择。
- 所有用户可见字符串都必须改用 `i18n` 调用，但为了减少一次性改动量，**第一阶段先保证 loader 与 install.sh 接入，再分批迁移 hy2.sh 字符串**。
- 完成后创建/使用 `feature/i18n-support` 分支，不提交 main。

- [ ] **Step 2: 在 hy2.sh 顶部加入 i18n loader 函数组**

添加位置：`server/hy2.sh` 第 2 行变量 `hihyV` 之后。

代码：

```bash
# i18n 多语言支持
HIHY_I18N_SCHEMA=1
HIHY_I18N_DIR="${HIHY_I18N_DIR:-/etc/hihy/i18n}"
HIHY_I18N_CONF="${HIHY_I18N_CONF:-/etc/hihy/conf/i18n.conf}"
HIHY_LANG="${HIHY_LANG:-}"

loadPersistedLanguage() {
    if [ -f "$HIHY_I18N_CONF" ] && [ -z "$HIHY_LANG" ]; then
        HIHY_LANG=$(grep -E '^HIHY_LANG=' "$HIHY_I18N_CONF" | tail -n 1 | cut -d'=' -f2-)
    fi
    HIHY_LANG="${HIHY_LANG:-en}"
}

savePersistedLanguage() {
    mkdir -p "$(dirname "$HIHY_I18N_CONF")"
    printf 'HIHY_LANG=%s\n' "$HIHY_LANG" >"$HIHY_I18N_CONF"
}

detectRtlFromMeta() {
    local file="$1"
    grep '"rtl"' "$file" 2>/dev/null | grep -q 'true'
}

# 按 key 读取 JSON 字符串；返回空表示未找到
i18nValueFromFile() {
    local file="$1"
    local key="$2"
    if [ ! -f "$file" ]; then
        return
    fi
    local line
    line=$(grep -E "^\s*\"${key}\":\s*\"" "$file" | head -n 1)
    [ -z "$line" ] && return
    # 提取第一个 "..." 之后的内容
    line="${line#*: \"}"
    line="${line%\"*}"
    printf '%s' "$line"
}

i18nLookup() {
    local key="$1"
    local lang_file="${HIHY_I18N_DIR}/hy2.sh.${HIHY_LANG}.json"
    local en_file="${HIHY_I18N_DIR}/hy2.sh.en.json"
    local value

    value=$(i18nValueFromFile "$lang_file" "$key")
    if [ -n "$value" ]; then
        printf '%s' "$value"
        return 0
    fi

    if [ "$HIHY_LANG" != "en" ]; then
        value=$(i18nValueFromFile "$en_file" "$key")
        if [ -n "$value" ]; then
            printf '%s' "$value"
            return 0
        fi
    fi

    printf '%s' "$key"
}

# Usage: i18n <key> [arg1] [arg2] ...
i18n() {
    local key="$1"
    shift
    local template
    template=$(i18nLookup "$key")
    # 简单模板插值：%s, %d, %%, 依次用 bash printf 填充
    printf "$template" "$@"
}
```

注意： Persian RTL 处理暂不在这里实现，仅在读取 `fa.json` 时 `detectRtlFromMeta` 供后续使用。

- [ ] **Step 2: 解析命令行参数 `--lang xx` 和环境变量 `HIHY_LANG`**

在 `main` 函数开头加入：

```bash
parseLanguageOption() {
    while [ $# -gt 0 ]; do
        case "$1" in
            --lang=*)
                HIHY_LANG="${1#*=}"
                shift
                ;;
            --lang)
                shift
                HIHY_LANG="${1:-}"
                shift
                ;;
            *)
                shift
                ;;
        esac
    done
}

downloadI18nFile() {
    local lang="$1"
    local base_url="$2"
    mkdir -p "$HIHY_I18N_DIR"
    local url="${base_url}/server/i18n/${lang}.json"
    local out="${HIHY_I18N_DIR}/hy2.sh.${lang}.json"
    if command -v wget >/dev/null 2>&1; then
        wget -q --no-check-certificate -O "$out" "$url" 2>/dev/null
    elif command -v curl >/dev/null 2>&1; then
        curl -fsSL -o "$out" "$url" 2>/dev/null
    else
        return 1
    fi
}

promptLanguageSelection() {
    echo -e "\033[32mPlease select installation language / 请选择安装语言:"
    echo -e "\033[33m 1) English"
    echo -e " 2) 中文"
    echo -e " 3) فارسی"
    echo -e " 4) Русский"
    echo -ne "\033[32mLanguage (default: 1): \033[0m"
    read -r lang_choice
    case "${lang_choice:-1}" in
        2) HIHY_LANG="zh" ;;
        3) HIHY_LANG="fa" ;;
        4) HIHY_LANG="ru" ;;
        *) HIHY_LANG="en" ;;
    esac
}

persistLanguage() {
    mkdir -p "$(dirname "$HIHY_I18N_CONF")"
    printf 'HIHY_LANG=%s\n' "$HIHY_LANG" >"$HIHY_I18N_CONF"
}
```

i18nLookup() {
    local key="$1"
    local lang_file="${HIHY_I18N_DIR}/hy2.sh.${HIHY_LANG}.json"
    local en_file="${HIHY_I18N_DIR}/hy2.sh.en.json"
    local value

    if [ -f "$lang_file" ]; then
        local schema
        schema=$(getI18nSchemaVersion "$lang_file")

            printf '%s' "$key"
            return 0
        fi
    fi

- [ ] **Step 1: 实现 i18n-extract.sh**

```bash
#!/bin/bash
set -euo pipefail

SCRIPT_FILE="${1:-server/hy2.sh}"
OUT_FILE="${2:-server/i18n/en.json}"

keys=$(grep -oE 'i18n [a-zA-Z0-9_]+' "$SCRIPT_FILE" | sed 's/i18n //' | sort -u)

tmp=$(mktemp)
{
    printf '{\n  "_meta": {\n    "schema_version": 1,\n    "language": "en",\n    "rtl": false\n  }'
    while IFS= read -r k; do
        [ -z "$k" ] && continue
        printf ',\n  "%s": "%s"' "$k" "$k"
    done <<< "$keys"
    printf '\n}\n'
} > "$tmp"

mv "$tmp" "$OUT_FILE"
echo "Wrote $OUT_FILE with $(echo "$keys" | wc -l) keys."
```

使用 `python3 -m json.tool` 或 `jq` 格式化。

- [ ] **Step 2: 实现 i18n-validate.sh**

```bash
#!/bin/bash
set -euo pipefail

BASE="server/i18n/en.json"
[ ! -f "$BASE" ] && echo "missing $BASE" && exit 1

base_keys=$(python3 -c "import json; print('\n'.join(json.load(open('$BASE')).keys()))" | grep -v '^_meta$' | sort)

errors=0
for f in server/i18n/*.json; do
    [ "$f" = "$BASE" ] && continue
    keys=$(python3 -c "import json; print('\n'.join(json.load(open('$f')).keys()))" | grep -v '^_meta$' | sort)
    missing=$(comm -23 <(echo "$base_keys") <(echo "$keys"))
    extra=$(comm -13 <(echo "$base_keys") <(echo "$keys"))
    if [ -n "$missing" ] || [ -n "$extra" ]; then
        echo "VALIDATION ERROR: $f"
        [ -n "$missing" ] && echo "  missing keys:" && echo "$missing" | sed 's/^/    /'
        [ -n "$extra" ] && echo "  extra keys:" && echo "$extra" | sed 's/^/    /'
        errors=$((errors+1))
    fi
done

if [ "$errors" -eq 0 ]; then
    echo "All translation files are consistent."
else
    exit 1
fi
```
